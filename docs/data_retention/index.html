数据保留策略

LiteLLM 云​

目的​

本政策概述了 LiteLLM 云为管理客户数据的保留和删除而实施的要求及控制/程序。

政策​

对客户而言：

  1. 活跃账户
  • 在客户的账户处于活跃状态期间，将保留其数据。这包括提示、生成的内容、日志以及使用指标等。
  1. 自愿关闭账户
  • 当账户自愿关闭时，数据进入“过期”状态。
  • 过期的账户数据将在 30 天内被保留（可根据需要调整）。
  • 在此之后，该账户及其所有相关数据将从 LiteLLM 云系统中永久删除。
  • 希望自愿关闭账户的客户应在启动关闭过程之前手动下载或备份其数据（通过可用 API 完成）。
  1. 不可预见的暂停
  • 如果客户的账户因未支付费用或其他原因而被不可预见地暂停，将有 14 天（可根据需要调整）宽限期，在此期间账户将无法访问，但客户可以解决导致暂停的问题后重新开启。
  • 在宽限期结束后，如果问题仍未解决，则账户将关闭，并且数据进入“过期”状态。
  • 数据一旦处于“过期”状态，将在 30 天（可根据需要调整）之后被永久删除，除非有法律要求另有规定。
  1. 暂停账户的手动备份
  • 如果客户希望手动备份暂停账户中的数据，他们必须将账户恢复到正常状态（通过解决支付或政策违规问题），以重新获得界面/API 访问权限。
  • 在账户处于暂停状态下时，无法访问来自该账户的数据。
  • 从暂停状态开始后的 14 天内（可根据需要调整），如果未达成解决方案，则账户将关闭，并且数据遵循上述“过期”数据删除的时间线。
  1. 定制保留策略
  • 企业客户可以根据其特定的合规和业务需求配置定制的数据保留期限。
  • 可用的自定义选项包括：
    • 调整活跃数据的保留期限（0-365 天）
  • 自定义保留策略必须通过 LiteLLM 云仪表板或 API 配置。

记录保护​

  • LiteLLM 云采取措施确保其控制下的所有记录免受丢失、破坏、篡改以及未经授权访问或泄露的风险。这些措施与相关立法、监管、合同和业务义务保持一致。
  • 在与第三方 CSP 合作时，LiteLLM 云会请求全面的信息以了解 CSP 的安全机制，包括为 LiteLLM 云存储或处理的记录提供的保护措施。
  • 被 LiteLLM 云聘用的云服务提供商必须披露他们为自己代表 LiteLLM 云收集和存储的记录所采取的保护措施。
