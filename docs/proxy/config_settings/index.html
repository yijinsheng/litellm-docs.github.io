config_settings

This comprehensive list of environment variables provides a detailed overview of the configuration options available for setting up and customizing the LiteLLM proxy service. Here's a summary of key categories:

General Configuration‚Äã

  • LITELLM_HOSTED_UI: URL of the hosted UI.
  • LITELLM_MODE: Operating mode (e.g., production, development).
  • LITELLM_ENVIRONMENT: Environment name for logging services.

Logging and Monitoring‚Äã

  • LITELLM_LOG: Enable detailed logging.
  • LITELLM_LOG_FILE: File path to write LiteLLM logs.
  • LITELLM_SALT_KEY: Salt key for encryption in LiteLLM.
  • LITELLM_SSL_CIPHERS: SSL/TLS cipher configuration.
  • LITELLM_MASTER_KEY: Master key for proxy authentication.
  • LITELLM_LICENSE: License key for LiteLLM usage.

Rate Limiting and Retry‚Äã

  • LITELLM_RATE_LIMIT_WINDOW_SIZE: Rate limit window size.
  • INITIAL_RETRY_DELAY, JITTER: Initial delay and jitter factor for retries.
  • MAX_RETRY_DELAY: Maximum delay for retries.

API Keys and Endpoints‚Äã

  • OPENAI_API_KEY, HF_API_BASE, HUGGINGFACE_API_KEY: API keys for various services.
  • LITELLM_MODEL_COST_MAP_URL: URL for model cost map data.
  • PROXY_BASE_URL: Base URL for the proxy service.

Secrets Management‚Äã

  • AWS_KMS_RESOURCE_NAME, GOOGLE_SECRET_MANAGER_PROJECT_ID, HCP_VAULT_ADDR: Names and IDs for secret managers.
  • LITELLM_SECRET_AWS_KMS_LITELLM_LICENSE: AWS KMS encrypted license key.

Email Configuration‚Äã

  • EMAIL_BUDGET_ALERT_TTL, EMAIL_SUPPORT_CONTACT, EMAIL_SIGNATURE: Email-related settings.

Database and Storage‚Äã

  • REDIS_HOST, REDIS_PORT, QDRANT_URL: Redis and Qdrant database URLs.
  • PROMETHEUS_URL: URL for Prometheus service.
  • SPEND_LOGS_URL: URL for retrieving spend logs.

Security and Authentication‚Äã

  • JWT_PUBLIC_KEY_URL, LITELLM_DONT_SHOW_FEEDBACK_BOX: JWT public key URL, feedback box settings.
  • ZSCALER_AI_GUARD_API_KEY, ZSCALER_AI_GUARD_POLICY_ID: Zscaler AI Guard API key and policy ID.

Customization and Overrides‚Äã

  • DOCS_URL, REDOC_URL: URLs for documentation and Redoc.
  • NO_DOCS, NO_REDOC: Flags to disable Swagger UI and Redoc.

Performance Tuning‚Äã

  • MAX_SIZE_PER_ITEM_IN_MEMORY_CACHE_IN_KB: Maximum size per item in memory cache.
  • COROUTINE_CHECKER_MAX_SIZE_IN_MEMORY: Maximum size for CoroutineChecker in-memory cache.

Debugging and Troubleshooting‚Äã

  • LITELLM_PRINT_STANDARD_LOGGING_PAYLOAD: Print standard logging payload to console.
  • MAX_STRING_LENGTH_PROMPT_IN_DB: Maximum length for strings in spend logs.

Multi-Instance Rate Limiting‚Äã

  • EXPERIMENTAL_MULTI_INSTANCE_RATE_LIMITING, LEGACY_MULTI_INSTANCE_RATE_LIMITING: Flags for rate limiting.

This list covers a wide range of configurations, from basic setup to advanced security and performance tuning. Each variable serves a specific purpose, allowing you to fine-tune the behavior of LiteLLM according to your needs.
