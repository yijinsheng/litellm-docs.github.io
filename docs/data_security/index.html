数据隐私与安全

在LiteLLM，保护您的数据隐私和安全是我们最优先考虑的事情。我们深知您分享给我们的数据的重要性，并会以最高标准的谨慎来处理这些数据。

通过LiteLLM Cloud，我们负责：

  • 部署
  • 扩展
  • 升级和安全补丁
  • 确保高可用性

安全措施​

LiteLLM Cloud​

  • 我们使用您的 LITELLM_MASTER_KEY 加密所有存储的数据，并在传输过程中使用 TLS 进行加密。
  • 我们的数据库和应用程序运行在 GCP 和 AWS 基础设施上，部分由 NeonDB 管理。
    • 北美数据区域：加利福尼亚北部（AWS/GCP us-west-1）及弗吉尼亚（AWS us-east-1）
    • 欧洲数据区域：德国/法兰克福（AWS/GCP eu-central-1）
  • 所有用户都可以通过 OAuth 2.0 的 Google、Okta、Microsoft 和 KeyCloak 进行单点登录 (SSO)。
  • 审计日志并设有保留策略
  • 控制可以访问您云中 LiteLLM 实例的允许 IP 地址

自托管 LiteLLM 实例​

  • 当您自托管时，LiteLLM 服务器不会存储任何数据或遥测数据
  • 有关安装和配置，请参见：自托管指南
  • 遥测：当您自托管 LiteLLM 时，我们不运行任何遥测

如有安全问题，请联系 support@berri.ai

LiteLLM Cloud 支持的数据区域​

LiteLLM 支持以下数据区域：

  • 北美，加利福尼亚北部（AWS/GCP us-west-1）
  • 欧洲，法兰克福，德国（AWS/GCP eu-central-1）

这两个区域之间的所有数据、用户账户和基础设施都是完全分离的

个人数据收集​

自托管 LiteLLM 用户：​

  • 当您自托管我们的软件时，不会收集或传输任何个人数据到 LiteLLM 服务器。
  • 所有生成或处理的数据均保留在您的自有基础设施中。

LiteLLM Cloud 用户：​

  • LiteLLM Cloud 跟踪语言模型的使用数据 - 我们不会访问或存储您 API 请求和响应的消息/回复内容。您可以查看 跟踪字段

如何使用和分享个人数据

  • 仅代理管理员可以查看其使用数据，并且只能看到他们组织的使用数据。
  • 代理管理员可以邀请其他用户/管理员访问他们的服务器以查看自己的使用数据
  • LiteLLM Cloud 不会将任何使用数据出售或与第三方共享。

Cookie 信息、安全和隐私​

自托管 LiteLLM 用户：​

  • Cookie 数据保留在您的自有基础设施中。
  • LiteLLM 使用少量的 cookie，仅用于允许代理用户访问 LiteLLM 管理界面。
  • 这些 cookie 存储在您登录后的浏览器中。
  • 我们不使用 cookie 用于广告、跟踪或任何其他目的，仅用于保持您的登录会话。
  • 所有用于维持应用程序 UI 用户认证和会话管理的 cookie 都是必需的。
  • 会话 cookie 在关闭浏览器、注销或 24 小时后过期。
  • LiteLLM 不使用第三方 cookie。
  • 管理界面通过 cookie 认证您的登录会话。
  • 该 cookie 存储为 JWT 并不可被系统其他部分访问。
  • 我们（LiteLLM）不会出于任何其他目的访问或分享此 cookie 数据。

LiteLLM Cloud 用户：​

  • LiteLLM 使用少量的 cookie，仅用于允许代理用户访问 LiteLLM 管理界面。
  • 这些 cookie 存储在您登录后的浏览器中。
  • 我们不使用 cookie 用于广告、跟踪或任何其他目的，仅用于保持您的登录会话。
  • 所有用于维持应用程序 UI 用户认证和会话管理的 cookie 都是必需的。
  • 会话 cookie 在关闭浏览器、注销或 24 小时后过期。
  • LiteLLM 不使用第三方 cookie。
  • 管理界面通过 cookie 认证您的登录会话。
  • 该 cookie 存储为 JWT 并不可被系统其他部分访问。
  • 我们（LiteLLM）不会出于任何其他目的访问或分享此 cookie 数据。

安全漏洞报告指南​

我们重视安全社区在保护我们的系统和用户方面的作用。如需报告安全漏洞，请：

  • 通过 support@berri.ai 发送详细信息
  • 包括重现问题的步骤
  • 提供任何相关附加信息

我们将尽快审核所有报告。请注意，目前我们不提供漏洞赏金计划。

安全漏洞扫描​

  • LiteLLM 在所有构建的 Docker 镜像上运行了 grype 安全扫描。
    • 请参见 grype litellm 检查在 ci/cd 中
    • 当前状态：✅ 通过。未发现高/关键级别漏洞。
